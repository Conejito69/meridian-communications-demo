# Sample Data

All data in this org is generated by
[`../../scripts/apex/generate-fictitious-data.apex`](../../scripts/apex/generate-fictitious-data.apex)
and
[`../../scripts/apex/create-sample-users.apex`](../../scripts/apex/create-sample-users.apex).
Nothing here was typed in by hand or copied from anywhere — every name,
company, address, and dollar figure is synthetic, generated by the scripts
themselves. See `../deployment/README.md` for how to run them.

## Why the discount and SLA numbers are deliberately skewed

Realistic demo data isn't just volume — it needs to actually exercise the
automation, or the dashboards and flows have nothing to show. Specifically:

- **~1 in 6 order lines exceeds the 20% discount threshold**, so
  `Service_Order_Discount_Approval_Router` and the `Discount_Approval`
  approval process have real records to act on, not a hypothetical.
- **A subset of Service Outage cases get an `SLA_Target_DateTime__c` set in
  the past**, so `SLA_Breach_Monitor` has genuine breaches to escalate the
  first time it runs, rather than an empty result set.
- **Region and account type are set consistently** with what the sharing
  rules filter on (`Type = Business Customer`, `Region__c` = North/South),
  so the Regional Sales sharing rules visibly do something.

## Fictional-world details

- Regions: "Cape Aldren" (North) and "Verrin Coast" (South) — invented
  place names, not real geography.
- Country: "Kestrelia" — invented.
- All emails use the `.invalid` TLD (reserved by RFC 2606 for exactly this
  purpose — guaranteed never to resolve to a real mailbox).
